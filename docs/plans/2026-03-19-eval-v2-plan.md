# Eval v2 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace Eval v1's fixed-split, coarse-threshold scoring with an Eval v2 pipeline centered on multi-delay retention, selectivity, and CV-first acceptance.

**Architecture:** Extend the existing evaluator rather than replacing the whole simulation stack. Add new score components in `Evaluator`, teach the cross-validator to aggregate Eval v2 outputs, then add a small driver that evaluates the current benchmark trio (`exp_0000`, `exp_0315`, `exp_0338`) under the new protocol. Keep Eval v1 fields available during transition so old reports do not break immediately.

**Tech Stack:** Python, numpy, existing `memory_decay` evaluator/runner/cross-validator modules, pytest
